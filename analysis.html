Human Activity Recognition Report

Author: Joshika

Introduction

This report aims to predict the manner in which subjects performed barbell lifts using accelerometer data from wearable sensors. The data was collected using devices like Jawbone Up, Nike FuelBand, and Fitbit. Our objective is to classify the activity quality based on the classe variable in the training dataset.

Background

Devices like Jawbone Up and Fitbit are part of the quantified self movement, helping individuals monitor personal activity. However, while many users measure how much they move, fewer measure how well they move. In this project, data was collected from sensors placed on six individuals performing barbell lifts in five different ways.

Dataset Source: PUC-Rio HAR Dataset

Data Loading and Cleaning


- Loaded 'pml-training.csv' and 'pml-testing.csv'
- Removed non-predictive variables: X, user_name, raw_timestamp_part_1, raw_timestamp_part_2, cvtd_timestamp, new_window, num_window
- Removed columns with >90% missing values
- Final features used for model: 53

Model Training


- Used caret package to build model
- Algorithm: Random Forest
- Method: 3-fold cross-validation
- Data split: 70% training / 30% validation

Model Evaluation


Confusion Matrix (Validation Set):
Accuracy: 0.9940
Out-of-sample error ≈ 1 - 0.9940 = 0.006 (0.6%)

Justification of Model Choice

Random Forest was selected due to its robustness, high accuracy, and ability to handle missing data and noisy variables. It performs internal feature selection and resists overfitting well, especially with high-dimensional datasets like ours.

Cross-validation Explanation

We employed 3-fold cross-validation using caret’s trainControl to ensure the model generalizes well and avoids overfitting. This approach divides the training set into three parts, trains on two, and validates on the third in rotation.
